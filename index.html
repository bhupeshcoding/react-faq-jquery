React FAQ

What is React?

React is a JavaScript library for building user interfaces, particularly for single-page applications. It allows you to create reusable UI components and efficiently update them when data changes

What is JSX?

JSX stands for JavaScript XML. It's a syntax extension that allows you to write HTML-like structures within your JavaScript code. This makes it easier to visualize the UI you're building.

What is the Virtual DOM?

The Virtual DOM is a lightweight in-memory representation of the actual DOM. When data changes in your React application, React first updates the Virtual DOM, then compares it to the actual DOM, and only updates the parts of the actual DOM that have changed. This makes updates more efficient.

What are components?

Components are the building blocks of React applications. They are reusable pieces of code that encapsulate a specific part of the UI. Components can be simple or complex, and they can be composed together to create more complex UIs.    

What are props?

Props are a way to pass data from a parent component to a child component. They are read-only and cannot be modified by the child component.

What is state?

State is a way to store data that is internal to a component. When the state changes, the component re-renders.  

What are React Hooks?

Hooks are functions that let you "hook into" React state and lifecycle features from functional components. They were introduced in React 16.8 and provide a more concise and functional way to manage state and side effects.        

What is Context API?

The Context API is a way to share data between components without having to pass props down through the entire component tree.

What are Higher-Order Components (HOCs)?

HOCs are functions that take a component and return a new component with added functionality. They are a powerful way to reuse component logic.

What is React Router?

React Router is a library that allows you to handle navigation and routing in your React application. It enables you to create single-page applications with multiple views.

What is server-side rendering (SSR)?

SSR is a technique for rendering React components on the server and sending the pre-rendered HTML to the client. This can improve initial load time and SEO. C

Common Interview Questions

What are the advantages of using React?

  • Component-based architecture: Reusability, maintainability, and testability.
  • Virtual DOM: Efficient updates and improved performance.
  • JSX: Easier to write and understand UI code.
  • Large community and ecosystem: Plenty of resources, libraries, and support.

What are the limitations of React?

  • Steep learning curve: Can be challenging to learn initially, especially JSX and component lifecycle.  
  • Focus on UI: React is primarily a UI library and may require additional libraries for routing, state management, etc.
  • Explain the difference between props and state. Props are read-only data passed from parent to child, while state is internal data managed within a component.

How do you handle forms in React?

React provides controlled components, where form data is handled in the component's state, and uncontrolled components, where form data is handled by the DOM.

What are the different lifecycle methods in React?

Lifecycle methods are special methods that get called during different phases of a component's life, such as mounting, updating, and unmounting. (Note: With the introduction of Hooks, some lifecycle methods are less relevant, but understanding the component lifecycle is still important.)
